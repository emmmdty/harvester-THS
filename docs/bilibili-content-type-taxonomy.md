# B站内容分类维护文档

Prompt 版本：`bilibili-taxonomy-v2026-06-12`

> 注意：本文档不被程序直接读取。B站分类实际运行以 `src/ai/platform-taxonomies.mjs`、`src/ai/content-classification.mjs`、`src/daily-records.mjs` 中的枚举、prompt、校验和下拉值为准。只修改本文档不会改变最终写入飞书的 `内容类型`；修改分类口径时必须同步更新源码和测试。

本文档说明 B站内容分类维护口径，以及 MiniMax 多模态分类和 DeepSeek 文本兜底分类的维护参考；实际运行源见上方说明。B站暂不做二级分类，只输出单层分类；通用 JSON 中 `secondaryType` 固定为空字符串。

## 输出规则

- `platform` 固定为 `bilibili`。
- `primaryType` 只能是：`采访内容`、`大佬生平`、`新手教学指标教学`、`海外搬运`、`短视频`。
- `secondaryType` 固定为空字符串。
- 模型不得创造新标签；不确定、证据冲突或低置信时，`reviewStatus` 写 `需人工复核`。
- 默认优先使用 MiniMax 多模态分类，结合标题、tag、作品类型、ASR、OCR、抽帧和封面图判断；只有素材无法获取、素材处理失败或 MiniMax API 不可用时，才降级到 DeepSeek 文本兜底。
- DeepSeek 文本兜底只使用标题和 tag；如果标题和 tag 不足以支持分类，必须返回 `需人工复核`，不要长期用文本兜底替代多模态判断。

统一 JSON：

```json
{
  "platform": "bilibili",
  "primaryType": "",
  "secondaryType": "",
  "confidence": 0,
  "reason": "",
  "evidence": [],
  "assetSignals": [],
  "reviewStatus": "通过"
}
```

## 易混边界

- `采访内容` vs `大佬生平`：重点是访谈、对话、个人投资经历分享时归 `采访内容`；重点是人物成长史、企业史、财富史、接班故事时归 `大佬生平`。
- `采访内容` vs `海外搬运`：如果标题明确是海外人物或海外采访，优先归 `海外搬运`；若没有海外线索，只是本土人物访谈，归 `采访内容`。
- `新手教学指标教学` vs 其他人物类：核心讲方法、指标、K线、盘面判断时归 `新手教学指标教学`；即使标题出现某位投资者，只要主体是生平或访谈，不归教学。
- `短视频` vs 前四类：`短视频` 是轻量内容兜底标签；如果标题、tag 或素材明确是访谈、人物生平、指标教学或海外搬运，优先归对应前四类，不因时长短而改成 `短视频`。
- B站不做二级分类。即使内容里有更细主题，`secondaryType` 也必须为空字符串。

## 人工反馈列

人工纠正直接维护在飞书表，不先做独立后台。建议在 AI 输出列后追加以下列：

| 列名 | 可选值/填写方式 | 用途 |
| --- | --- | --- |
| 人工分类状态 | `正确`、`已纠正`、`边界待定`、`不纳入样本` | 标识这条 AI 分类能否进入反馈样本。 |
| 人工一级类型 | B站一级短标签 | `已纠正` 时必填；`正确` 时可留空。 |
| 人工二级类型 | 固定留空 | B站暂不做二级分类。 |
| 人工备注 | 自由文本 | 记录错误原因、边界说明或不纳入原因。 |

处理规则：

- `正确`：AI 标签可信，可作为金标样本。
- `已纠正`：人工一级覆盖 AI 标签，可作为错误样本和金标样本。
- `边界待定`：暂不进入 prompt 和评测集，等业务定口径。
- `不纳入样本`：素材缺失、标题污染、重复异常、非正常业务内容，不进入反馈样本。

## Prompt 反馈进化流程

1. 每周导出人工反馈行，保留标题、tag、素材证据、AI 输出、人工输出、人工备注、Prompt 版本。
2. 将 `正确` 和 `已纠正` 行组成回放集；`边界待定` 和 `不纳入样本` 不参与回放。
3. 同类错误累计 3 条以上，或影响关键业务分类时，才修改分类标准、边界规则或少样本示例。
4. 修改 Prompt 前后必须用同一批回放集对比：总准确率、一级准确率、低置信复核率、旧样本回归错误。
5. 版本号按日期更新，例如 `bilibili-taxonomy-v2026-06-12`；变更记录必须写清：变更原因、采纳样本、修改内容、回放结果、是否影响旧分类。

## 后续新增分类流程

1. 收集至少 5 条代表样本，包含标题、tag、素材证据和期望分类。
2. 写清新增标签定义、判断线索、正例、反例和易混边界。
3. 检查是否能并入现有标签；只有现有标签无法稳定覆盖时才新增。
4. 更新本文档分类表和对应 Prompt 版本。
5. 回放旧样本，确认新增标签没有大量抢占旧标签。
6. 工程实施阶段再同步更新下拉选项、校验逻辑和测试。
