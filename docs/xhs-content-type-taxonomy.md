# 小红书内容分类维护文档

Prompt 版本：`xhs-taxonomy-v2026-06-10`

本标准用于小红书内容分类维护、MiniMax 多模态分类和 DeepSeek 文本兜底分类。分类标签采用短标签：输出字段不写括号、斜杠和“类”字，解释写在本文档中。

## 输出规则

- `platform` 固定为 `xhs`。
- `primaryType` 只能是：`图文`、`视频`。
- `secondaryType` 必须从对应一级类型下选择。
- 图文二级只能是：`财富人物`、`理财方法`、`行业盘点`、`互动话题`。
- 视频二级只能是：`股友说`、`社区话题`、`资讯`、`说唱`、`段子`、`长视频`。
- 模型不得创造新标签；不确定、证据冲突或低置信时，`reviewStatus` 写 `需人工复核`。
- 默认优先使用 MiniMax 多模态分类，结合标题、tag、作品类型、图片、OCR、视频抽帧和 ASR 判断；只有素材无法获取、素材处理失败或 MiniMax API 不可用时，才降级到 DeepSeek 文本兜底。
- 小红书一级类型是 `图文` / `视频`，必须优先由素材形态或明确作品类型决定；只有标题和 tag、且没有作品类型或素材证据时，不得硬猜一级类型，应返回 `需人工复核`。

统一 JSON：

```json
{
  "platform": "xhs",
  "primaryType": "",
  "secondaryType": "",
  "confidence": 0,
  "reason": "",
  "evidence": [],
  "assetSignals": [],
  "reviewStatus": "通过"
}
```

## 易混边界

- `图文` vs `视频`：优先使用作品类型或素材形态判断。只有标题和 tag 且缺少作品类型/素材证据时，不得强行区分，返回 `需人工复核`。
- `财富人物` vs `理财方法`：核心讲人物经历、财富曲线和投资大佬时归 `财富人物`；核心讲方法、思维、书单、K线或成长路径时归 `理财方法`。
- `行业盘点` vs `互动话题`：核心是产业链、板块、资金、ETF、涨停复盘等信息梳理时归 `行业盘点`；核心是提问、选择、评论互动时归 `互动话题`。
- 视频 `资讯` 是兜底视频内容标签。若标题/tag 明确是 `股友说`、`社区话题`、`说唱`、`段子` 或 `长视频`，不要归为 `资讯`。

## MiniMax 多模态提示词

适用场景：默认优先使用。已拿到图文图片、视频抽帧、OCR、ASR 或完整素材时，必须优先走本提示词。只有素材无法获取、素材处理失败或 MiniMax API 不可用时，才降级到 DeepSeek 文本兜底提示词。

```text
你是小红书内容分类助手。
唯一分类依据是用户提供的本地分类标准，不允许引入外部平台垂类口径。
请先判断 primaryType，再只在该 primaryType 下选择 secondaryType。
只能返回 JSON，不要返回 Markdown、解释或额外文本。

输出 JSON：
{
  "platform": "xhs",
  "primaryType": "",
  "secondaryType": "",
  "confidence": 0,
  "reason": "",
  "evidence": [],
  "assetSignals": [],
  "reviewStatus": "通过|需人工复核"
}

字段规则：
- primaryType 只能是：图文、视频。
- 图文 secondaryType 只能是：财富人物、理财方法、行业盘点、互动话题。
- 视频 secondaryType 只能是：股友说、社区话题、资讯、说唱、段子、长视频。
- primaryType 必须优先根据作品类型或素材形态判断；如果无法确认图文/视频，primaryType 和 secondaryType 返回空字符串，reviewStatus 写 需人工复核。
- evidence 引用标题、tag、ASR、OCR 或画面中的短片段。
- assetSignals 写画面/音频/图文素材提供的判断线索；没有则返回空数组。
- confidence 低于 0.75、证据冲突、素材看不清或标签边界不确定时，reviewStatus 写 需人工复核。

输入字段：
- 标题：{{title}}
- tag：{{tags}}
- 作品类型：{{itemType}}
- ASR：{{asrText}}
- OCR：{{ocrText}}
- 抽帧/图片：随消息提供

使用本文档“分类表”和“易混边界”。
```

## 人工反馈列

人工纠正直接维护在飞书表，不先做独立后台。建议在 AI 输出列后追加以下列：

| 列名 | 可选值/填写方式 | 用途 |
| --- | --- | --- |
| 人工分类状态 | `正确`、`已纠正`、`边界待定`、`不纳入样本` | 标识这条 AI 分类能否进入反馈样本。 |
| 人工一级类型 | `图文` 或 `视频` | `已纠正` 时必填；`正确` 时可留空。 |
| 人工二级类型 | 对应一级下的二级短标签 | `已纠正` 时按最终判断填写。 |
| 人工备注 | 自由文本 | 记录错误原因、边界说明或不纳入原因。 |

处理规则：

- `正确`：AI 标签可信，可作为金标样本。
- `已纠正`：人工一级/二级覆盖 AI 标签，可作为错误样本和金标样本。
- `边界待定`：暂不进入 prompt 和评测集，等业务定口径。
- `不纳入样本`：素材缺失、标题污染、重复异常、非正常业务内容，不进入反馈样本。

## Prompt 反馈进化流程

1. 每周导出人工反馈行，保留标题、tag、素材证据、AI 输出、人工输出、人工备注、Prompt 版本。
2. 将 `正确` 和 `已纠正` 行组成回放集；`边界待定` 和 `不纳入样本` 不参与回放。
3. 同类错误累计 3 条以上，或影响关键业务分类时，才修改分类标准、边界规则或少样本示例。
4. 修改 Prompt 前后必须用同一批回放集对比：总准确率、一级准确率、二级准确率、低置信复核率、旧样本回归错误。
5. 版本号按日期更新，例如 `xhs-taxonomy-v2026-06-10`；变更记录必须写清：变更原因、采纳样本、修改内容、回放结果、是否影响旧分类。

## 后续新增分类流程

1. 收集至少 5 条代表样本，包含标题、tag、素材证据和期望分类。
2. 写清新增标签定义、判断线索、正例、反例和易混边界。
3. 检查是否能并入现有标签；只有现有标签无法稳定覆盖时才新增。
4. 更新本文档分类表和对应 Prompt 版本。
5. 回放旧样本，确认新增标签没有大量抢占旧标签。
6. 工程实施阶段再同步更新下拉选项、校验逻辑和测试。
