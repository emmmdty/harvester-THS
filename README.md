# 作品采集器

这是给同事使用的作品采集工具。解压后双击启动，网页登录三个平台账号后即可采集。

## 使用步骤

### 1. 双击启动

macOS 双击：

```text
启动作品采集面板.command
```

Windows 双击：

```text
启动作品采集面板.cmd
```

启动窗口会自动检查运行环境、安装缺失组件，并打开网页：

```text
http://127.0.0.1:3000/
```

如果 3000 端口已经被占用，启动脚本会自动改用 3001 到 3010 之间的可用端口，并打开对应网页。
默认按局域网模式启动；同事只需要使用自动打开的本机网页即可。
启动窗口不要关闭。关闭窗口后，采集器也会停止。

### 2. 扫码登录三个平台

网页打开后，依次完成三个平台登录：

1. 切换到“小红书”，点击“打开登录”，扫码登录，登录完成后关闭弹出的浏览器窗口。
2. 切换到“抖音”，点击“打开登录”，扫码登录，登录完成后关闭弹出的浏览器窗口。
3. 切换到“B站”，点击“打开登录”，扫码登录，登录完成后关闭弹出的浏览器窗口。

每个平台登录完成后，可以点击“检测登录”确认状态。

### 3. 开始采集

推荐使用“全渠道”：

1. 切换到“全渠道”。
2. 选择“全渠道日期”。
3. 点击“全渠道采集”。

采集器会自动采集小红书、抖音、B站，并写入已配置好的飞书表格。
采集过程中会把素材缓存到本机：视频优先使用下载器，抖音/小红书图文会使用已登录浏览器抓取页面素材或截图，用于后续 AI 多模态判断。

## 常见问题

- 网页打不开：确认启动窗口还开着。
- 启动时提示正在安装：等待自动完成即可，第一次启动可能需要几分钟。
- 安装失败：把启动窗口截图发给技术同事。
- 登录失败：重新点击对应平台的“打开登录”，扫码后再检测登录。
- 采集失败：把网页里的“运行日志”最后几行截图发给技术同事。

## 交付说明

- `.env` 已随交付包提供，不需要同事额外配置飞书。
- `platform-accounts.json` 已随交付包提供，不需要同事维护账号列表。
- 登录状态只保存在同事自己的电脑里，不随交付包分发。
- 不要删除 `.env`、`platform-accounts.json`、`package.json`、`package-lock.json`、`src/`、`public/` 或启动脚本。
